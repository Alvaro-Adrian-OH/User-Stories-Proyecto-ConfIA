Feature: EP06–Dashboard, Métricas y Reportes
  Como usuario académico o docente
  Quiero visualizar métricas de uso y acceder a reportes de actividad
  Para monitorear mi progreso investigativo y supervisar la calidad académica de los estudiantes

  # User Story 19

  @AT-19 @US-19 @dashboard
  Scenario: Métrica de fuentes validadas en el mes mostrada al cargar el dashboard
    Given que un usuario autenticado accede al dashboard de ConfIA
    When la vista se carga correctamente
    Then el sistema muestra la métrica "Fuentes validadas este mes" con el valor actualizado correspondiente a la actividad del usuario

  @AT-19 @US-19 @dashboard
  Scenario: Gráfica de progreso de calidad refleja los datos reales del usuario
    Given que el usuario autenticado visualiza el dashboard
    When la sección de progreso se renderiza
    Then la gráfica de "Progreso de calidad" refleja correctamente los datos de las validaciones realizadas por el usuario

  @AT-19 @US-19 @dashboard
  Scenario: Métricas en cero o vacías cuando no hay actividad en el mes actual
    Given que el usuario no tiene actividad registrada en el mes actual
    When el dashboard se carga
    Then las métricas muestran valores en cero o un estado vacío con un mensaje orientativo para el usuario

  @AT-19 @US-19 @dashboard
  Scenario: Métricas actualizadas tras una nueva validación
    Given que el usuario realiza una nueva validación de fuente
    When regresa al dashboard o lo recarga
    Then las métricas del dashboard se actualizan con los datos más recientes de la actividad del usuario

  @AT-19 @US-19 @dashboard
  Scenario: Datos del dashboard corresponden exclusivamente al usuario autenticado
    Given que el usuario autenticado accede al dashboard
    When visualiza las gráficas y métricas presentadas
    Then los datos mostrados corresponden exclusivamente a la cuenta del usuario autenticado sin mezclar información de otros usuarios

  # User Story 21

  @AT-21 @US-21 @reporte-docentes
  Scenario: Acceso al panel de docentes bloqueado con error 403 para usuarios con rol Alumno
    Given que un usuario con rol "Alumno" intenta acceder al panel de reportes para docentes
    When el sistema valida los permisos del usuario
    Then retorna un error HTTP 403 (Forbidden)
    And bloquea el acceso al panel por completo

  @AT-21 @US-21 @reporte-docentes
  Scenario: Docente visualiza fuentes validadas y observadas de sus estudiantes
    Given que un usuario con rol "Docente" accede al panel de reportes
    When la vista del panel se carga correctamente
    Then el sistema muestra la lista de estudiantes con sus respectivas fuentes validadas y observadas

  @AT-21 @US-21 @reporte-docentes
  Scenario Outline: Distinción visual entre estados de fuente en el reporte docente
    Given que el docente visualiza el reporte de actividad de un estudiante en el panel
    When revisa una fuente en estado "<estado>"
    Then el sistema la distingue visualmente como "<indicador_visual>"

    Examples:
      | estado      | indicador_visual        |
      | aprobada    | fuente aprobada          |
      | rechazada   | fuente rechazada         |
      | observación | fuente en observación    |

  @AT-21 @US-21 @reporte-docentes
  Scenario: Panel indica ausencia de actividad para estudiante sin validaciones
    Given que el docente accede al panel de reportes
    When revisa el registro de un estudiante que aún no ha realizado validaciones
    Then el sistema indica que no hay actividad registrada para ese usuario

  @AT-21 @US-21 @reporte-docentes
  Scenario Outline: Filtro individual aplicado correctamente en el panel docente
    Given que el docente aplica el filtro por "<tipo_filtro>" con el valor "<valor_filtro>"
    When el sistema filtra los datos según el criterio seleccionado
    Then retorna únicamente los registros que corresponden a "<criterio_esperado>"

    Examples:
      | tipo_filtro | valor_filtro           | criterio_esperado                          |
      | estudiante  | nombre específico      | el estudiante indicado                     |
      | fecha       | rango de fechas         | el rango de fechas indicado                |
